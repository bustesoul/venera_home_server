# `internal/` 目录说明

当前服务端实现改为按模块拆分的扁平目录：

- `app/`：核心应用模型、扫描、元数据合并、章节分页物化
- `archive/`：ZIP / RAR / 7Z / PDF 归档访问
- `backend/`：本地、SMB、WebDAV 存储后端
- `cli/`：命令行启动入口封装
- `config/`：配置加载与解析
- `favorites/`：收藏夹持久化
- `httpapi/`：HTTP 路由、媒体分发、页面缓存逻辑
- `shared/`：跨模块复用的小型工具函数
- `tests/`：独立测试模块与 `testkit/` 公共测试支撑

这套结构的目标是：

1. 入口层、传输层、领域层、基础设施层分开；
2. 目录按模块对齐，而不是单包堆文件；
3. 测试与生产代码物理隔离，但仍保持模块归属清晰。
