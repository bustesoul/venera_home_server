# Venera Home 使用指南

本文说明如何使用 `Venera Home Server + venera_home.js` 在 Venera 中阅读你自己已有的漫画库。

## 1. 方案组成

这套方案由两部分组成：

- `venera_home_server`：Go 写的本地服务端，负责扫描本地目录、SMB、WebDAV，并提供 HTTP API。
- `venera_home.js`：Venera 的漫画源配置文件，负责把服务端 API 映射成 Venera 可用的漫画源。

相关文件：

- `C:\Users\buste\dev\venera-configs\venera_home.js`
- `C:\Users\buste\dev\venera-configs\venera_home_server\venera_home.js`
- `C:\Users\buste\dev\venera-configs\venera_home_server\server.example.toml`
- `C:\Users\buste\dev\venera-configs\venera_home_server`

其中，根目录的 `venera_home.js` 用于提交到 `venera-configs`；`venera_home_server` 目录里的同名文件用于和后端一起维护。

## 2. 当前支持范围

当前可用能力：

- 本地目录漫画
- SMB 共享漫画（当前按 Windows 方案实现）
- WebDAV 漫画
- 图片目录格式漫画
- `cbz` / `zip` 漫画包
- `cbr` / `rar` 漫画包
- `7z` / `cb7` 漫画包
- `pdf` 漫画文件（当前使用 Windows 内建 PDF 渲染）
- 首页、分类、搜索、详情、章节阅读
- 收藏夹、多文件夹收藏
- 手动重扫
- `ComicInfo.xml` 与 `.venera.json` 元数据覆盖

当前未实现：

- 评论、签到、互联网账号登录
- 远程元数据抓取
- 非 Windows 平台上的 PDF 渲染

## 3. 漫画目录建议

推荐的本地结构：

### 结构 A：单本漫画目录

```text
D:\Comics\孤独摇滚\
  001.jpg
  002.jpg
  003.jpg
```

这会被识别成一本单章节漫画。

### 结构 B：多章节漫画目录

```text
D:\Comics\迷宫饭\
  01\
    001.jpg
    002.jpg
  02\
    001.jpg
    002.jpg
```

这会被识别成一个漫画，下面有多个章节。

### 结构 C：压缩包 / 文档漫画

```text
D:\Comics\Packed\
  monster.cbz
  another.zip
  legacy.cbr
  packed.cb7
  album.pdf
```

每个 `cbz` / `zip` / `cbr` / `rar` / `cb7` / `7z` / `pdf` 都会被识别成一本漫画；如果放在同一个系列目录下，也可以被识别成多章节。

## 4. 可选元数据文件

### `ComicInfo.xml`

如果目录或压缩包里带有 `ComicInfo.xml`，服务端会读取常见字段，例如：

- 标题
- 简介
- 作者
- 标签
- 语言

### `.venera.json`

你也可以手动放一个 `.venera.json` 覆盖元数据，例如：

```json
{
  "title": "迷宫饭",
  "subtitle": "九井谅子",
  "description": "手工维护的说明",
  "authors": ["九井谅子"],
  "tags": ["奇幻", "美食", "冒险"],
  "language": "zh"
}
```

优先级大致是：

1. `.venera.json`
2. `ComicInfo.xml`
3. 文件名 / 目录名推断

## 5. 配置服务端

编辑：

- `C:\Users\buste\dev\venera-configs\venera_home_server\server.example.toml`

### 本地目录示例

```toml
[[libraries]]
id = "local-main"
name = "Local Manga"
kind = "local"
root = "D:/Comics"
scan_mode = "auto"
```

### SMB 示例

```toml
[[libraries]]
id = "nas-manga"
name = "NAS Manga"
kind = "smb"
host = "192.168.1.10"
share = "Media"
path = "Comics"
username = "reader"
password_env = "SMB_PASS"
scan_mode = "auto"
```

运行前先设置环境变量，例如：

```powershell
$env:SMB_PASS = "你的密码"
```

### WebDAV 示例

```toml
[[libraries]]
id = "dav-manga"
name = "WebDAV Manga"
kind = "webdav"
url = "https://dav.example.com/remote.php/dav/files/user"
path = "/Comics"
username = "reader"
password_env = "WEBDAV_PASS"
scan_mode = "auto"
```

运行前先设置：

```powershell
$env:WEBDAV_PASS = "你的密码"
```

## 6. 启动服务端

在目录：

- `C:\Users\buste\dev\venera-configs\venera_home_server`

执行：

```powershell
go run . -config .\server.example.toml
```

默认监听：

- `http://127.0.0.1:34123`

如果你希望手机上的 Venera 访问电脑上的服务，不要用 `127.0.0.1`，而要用电脑的局域网地址，例如：

- `http://192.168.1.20:34123`

## 7. 在 Venera 中使用

### 方式一：直接导入 `venera_home.js`

把下面这个文件作为自定义源导入：

- `C:\Users\buste\dev\venera-configs\venera_home.js`

### 方式二：发布到你自己的可访问地址

如果你需要多设备使用，可以把 `venera_home.js` 放到你自己的 HTTP / CDN 地址，再在 Venera 中通过 URL 加载。

## 8. Venera 源设置说明

在 Venera 中打开该源后，主要设置项如下：

- `Server URL`
  - 填你的服务端地址，例如 `http://192.168.1.20:34123`
- `Token`
  - 填 `server.example.toml` 里的 `token`
- `Default Library ID`
  - 留空表示聚合所有书库
  - 填某个书库 ID，则默认只显示该书库
- `Default Sort`
  - 默认排序方式
- `Page Size`
  - 列表分页大小

建议：

- 单机自用：`Server URL = http://127.0.0.1:34123`
- 手机访问电脑服务：`Server URL = http://你的电脑局域网IP:34123`

## 9. 常见问题

### Q1：为什么测试连接失败？

常见原因：

- 服务端没有启动
- `Server URL` 填成了 `127.0.0.1`，但你实际是在手机上访问电脑服务
- `Token` 不一致
- SMB / WebDAV 密码环境变量没有设置
- 配置文件路径或根目录写错

### Q2：为什么扫描不到漫画？

先检查：

- 根目录是否真的有图片文件或受支持的漫画包（例如 `cbz/zip/rar/7z/pdf`）
- 目录层级是否太杂
- 如果是 `pdf`，当前服务端是否跑在 Windows 上

### Q3：为什么 SMB 库打不开？

当前 SMB 是按 Windows 方案实现的，适合你现在这台机器。如果以后你要在 Linux 上长期跑服务，建议优先用 WebDAV，或者后续再补跨平台 SMB 实现。

### Q4：为什么第一次打开压缩包或 PDF 较慢？

因为服务端会先把远程压缩包拉到本地缓存，再解析内容；如果是 PDF，还会在首次访问页面时渲染成 PNG。第一次慢一点是正常的，后续会复用缓存和渲染结果。

## 10. 推荐使用流程

1. 先只配置一个 `local` 书库，确认流程跑通
2. 在浏览器访问 `http://127.0.0.1:34123/api/v1/bootstrap` 确认服务已启动
3. 在 Venera 中填写 `Server URL` 和 `Token`
4. 点击 `Test Connection`
5. 能看到书库后，再逐步加入 SMB / WebDAV
6. 最后再按需整理 `ComicInfo.xml` 或 `.venera.json`
